Arrow functions and classes in javascript

November 6, 2017

JavaScript’s new features have been making the web development easier and provide clean code to developers that is being helpful for them. In this blog post, I will explain two new features arrow functions and classes in ES6 in detail. This blog post will be helpful for you.

Arrow Functions

ES6 has introduced a number of new features to make web development easy and better. Arrow functions provide concise syntax for code that make code writing easy. They have implicit returns and they do not rebind the value of this when you use arrow functions inside another function. Firstly, we will see simple function expression:

Var add = function (num1, num2) {

Var sum = num1 + num2;

Return sum;

}

Var sum = add (5,3);

Console.log(sum);

Function Expression with arrow functions:

ES6 came with concise syntax for writing a function expression that is arrow function. Example of function expression using the arrow function:

Var add = (num1, num2) => {return num1 + num2;};

In this example, you can see that use of arrow function has makes your code concise and easy.

In arrow function, parameters should be passed in small parenthesis. If your function expression has two parameters, then it will be as follows:

Var add = (num1, num2) => {return num1 + num2;};

If there is only one parameter going to be passed in the function expression, then there is no need of parenthesis. You can define parameters as below:

Var add = num => {return num*10;};

If there have no parameters in the function expression, then you have to use empty parenthesis. The code will give an error without empty parameters.

Arrow functions can be declared without the empty parenthesis and return statement. Let’s see the syntax for both.

Var sum = (a, b) => a + b; (without return statement)

Empty parenthesis:

Var sum = () => console.log (“hello world”);

Arrow function Features

Arrow function supports many features in JavaScript to write clean and useful code. Arrow functions support the use of different features that makes JavaScript best programming language for developers.

  • Returning object literals:

    Arrow function in JavaScript supports object literals. To get this feature in your code you only need to enclose a return object in small brackets. The example of function using object literals is below:

    Var person = (name, contact) => ({

Name: name,

Contact: contact

})

Var p = person (“kulwinder”, 6045678765);

Console.log(p);

As shown in this example, the object to be returned is enclosed in brackets. This is essential because if you don’t do this, JavaScript would not be able to differentiate between object literal and function body.

  • Arrow function supports rest parameters: 1

    The Arrow function in ES6 provides a feature of Rest parameters that allows to use any number of arguments as an array in a function. Let’s see the example to declare a function using rest parameters:

        Function add (...args) {

            var r = 0;
            for (let i=0; i<args. length; i++) {
            result = r + args[i];
    }
        return result; 
    }
    var r = add(5,6,4,2);
    console.log(r);
    var t = add(8,30,8);
    console.log(t);

    This example shows that programmers can pass any number of arguments to arrow function. The programmer can pass other parameters as well with rest parameters. It limits to write a lot of code again and again.

  • No separate this value: 2

    The arrow function does not have its own this value. The arrow function’s this value can be inherited from the enclosing scope. Let’s consider the following code to understand the concept of this value in arrow function:

    var person = {
name: ‘kulwinder’,
attribute: function () {
condole.log(this)
var property = () => {console.log(this)}
property ();
}
};

In the above code, the person is an object with one property and method that is named and attribute. Here I created an arrow function property with this value that is not this value of arrow function. This value is being inherited from function attribute.

Classes in ES6

ES6 has provided the feature of classes in the new release. Classes are the syntactical way to work with objects and prototypes. They provide a clean and clear syntax for creating objects and deal with inheritance.

Class food {
Constructor (calorie, protein)
{
this. calorie = calorie;
this. Protein = protein
}
}

Classes are special function and it has two components class expression and class declaration as functions.

Extending the classes: 3

Classes can be extended using extend keyword. Programmers can access the class methods and members in another class by extending the class. This provides the concise code that means you does not need to define those methods in another class you can simply call one class in another.

Let’s see the example of extend the class below:

Class food {
Constructor (calorie, protein)
    {
    this. calorie = calorie;
    this. Protein = protein
    }
}


Class apple extends food {
{
    Console.log (“calories in apple:”);
}

In this example, class apple can access the methods and members of food class that limit the repetition of code.

Static methods in classes:

The static keyword is used to define the static methods in a class. The static methods cannot be called by an instance of the class. They are used to create a utility function for an application.

Example:

Class person {  

    int static name;  

    int static age;  

    static attribute (name, age)  

    {  

        Var name= name;  

        Var age = age;  

    }  

}
